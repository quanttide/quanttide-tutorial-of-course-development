# 如何设计和制作场景？

一个场景呢，就是一个最小操作单元，通常由一系列步骤组成，比如说像这个配置密钥，那么就是点开 Z 的配置，然后找到 DeepSeek 的位置，然后填上，然后结束。

这样的一个操作单元大概是两分钟左右，因为在实际过程中，一个步骤可能是几秒到十几秒。

从录制的实际情况来看，完整地录制一个异常场景，相比于录制一个异常步骤要容易很多。同时，对于学生来讲接受也会更容易，对于平台维护来讲也会更容易维护。

所以，我们就以场景作为一个最小单元。

场景也可以是起点。比如我们在日常协作的过程中发现，大家对于 Z 的文件数怎么使用不太熟悉，比如：
1. 它和文件系统之间的关系是什么样子的；
2. 要怎么去拖拽；
3. 怎么把它拖拽到这个 Agent 等等。

像这些的话，就可以是一个或者是多个场景。在一个 Agent 的科普视频之中，它可能就会。
